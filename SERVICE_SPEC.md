# Lane Changer — 서비스 명세서

> 최근 전적을 분석해서 플레이어에게 딱 맞는 포지션과 챔피언을 추천하는 서비스

---

## 1. 서비스 개요

| 항목 | 내용 |
|------|------|
| 서비스명 | Lane Changer |
| 한 줄 설명 | 리그 오브 레전드 전적 기반 포지션 추천 서비스 |
| 핵심 가치 | 감이 아닌 데이터로, 팩트만 말하는 냉혹한 코치 |
| 타깃 유저 | 포지션 변경을 고려 중인 LoL 유저 |
| 배포 URL | https://hobby-lol.vercel.app (예정) |
| 기술 스택 | Next.js, Vercel, Riot Games API, Claude API |

---

## 2. 타깃 유저

- 현재 포지션에서 슬럼프를 겪고 있는 유저
- 새로운 포지션에 도전하고 싶지만 뭘 해야 할지 모르는 유저
- 친구의 전적을 분석해서 팩폭하고 싶은 유저

---

## 3. 핵심 기능

| 기능 | 설명 |
|------|------|
| 전적 분석 | Riot API로 최근 20게임 자동 수집 |
| 포지션 점수 계산 | KDA, CS, 시야점수, CC, 딜량, 오브젝트 등 종합 분석 |
| 포지션 추천 | 주포지션 제외 4개 포지션 적합도 점수 표시 |
| 챔피언 추천 | 유저 플레이스타일 태그 기반 개인화 추천 |
| AI 팩폭 코멘트 | Claude API로 생성하는 냉혹한 반말 코치 코멘트 |
| 최근 검색 기록 | localStorage 기반 드롭다운 (최대 5개) |

---

## 4. 화면 플로우

### 4-1. 전체 플로우

```
[메인 화면]
    │
    ├── 소환사명 입력 (gameName#태그)
    │       └── 최근 검색 드롭다운 표시
    │
    ▼
[로딩 화면]
    │ Riot API 호출 → 분석 → Claude API 코멘트 생성
    ▼
[결과 화면]
    │
    ├── 소환사 요약 카드
    │       ├── 소환사명 / 태그
    │       ├── 분석 게임 수 / 승률 / 주 포지션
    │       ├── 플레이스타일 태그
    │       └── AI 팩폭 코멘트
    │
    └── 추천 포지션 카드 (케이스별 상이)
            ├── 포지션명 / 적합도 점수 / 점수 바
            ├── 추천 이유 목록
            └── 추천 챔피언 3개 (이미지 + 이유)
```

## 5. 케이스별 결과 화면 분기

### Case 1. 주포지션 명확 + 승률 55% 미만 (일반)
- **팩폭 톤**: 현재 포지션이 맞지 않는다고 냉혹하게 팩폭
- **추천 헤더**: "추천 포지션"
- **표시 개수**: 4개 포지션 전부 점수 표시
- **점수 범위**: 0~100점

### Case 2. 주포지션 명확 + 승률 55% 이상 (고승률)
- **팩폭 톤**: 지금 포지션 잘 하고 있는데 굳이 왜 바꾸냐고 팩폭
- **추천 헤더**: "굳이 바꾸고 싶다면"
- **표시 개수**: 1개만 표시
- **점수 범위**: 0~50점 (고승률 신호로 의도적으로 낮게 표시)

### Case 3. 주포지션 없음 (멀티 포지션, 특정 포지션 40% 미만)
- **팩폭 톤**: 주포지션도 없이 기웃거린다고 팩폭, 하나만 파라고 단호하게
- **추천 헤더**: "당장 이것만 파"
- **표시 개수**: 1개만 표시
- **주 포지션 표기**: "주포지션 없음" (빨간색)

## 6. 화면별 상세 명세

### 6-1. 메인 화면 (검색 전)

| 요소 | 내용 |
|------|------|
| 헤더 | Lane Changer 로고 (클릭 시 검색 초기화) |
| 히어로 타이틀 | "새로운 라인을 찾아보세요" |
| 서브 카피 | "최근 전적을 분석해서 당신에게 딱 맞는 포지션과 챔피언을 추천해드려요" |
| 검색 입력창 | `소환사명#태그` 형식, 태그 없으면 KR1 자동 적용 |
| 최근 검색 드롭다운 | 포커스 시 표시, 최대 5개, 클릭 시 자동 검색 |
| 분석하기 버튼 | Enter 키 또는 버튼 클릭으로 분석 시작 |

### 6-2. 로딩 화면

| 요소 | 내용 |
|------|------|
| 스켈레톤 UI | 3개 shimmer 블록 표시 |
| 로딩 문구 | "최근 20게임을 분석하고 있어요..." |
| 소요 시간 | 약 5~10초 (Riot API 배치 호출 + Claude API) |

### 6-3. 결과 화면 — 소환사 요약 카드

| 요소 | 내용 |
|------|------|
| 소환사명 | gameName + #tagLine |
| 분석 요약 | 게임 수 / 승률 (파란색: 50%↑, 빨간색: 50%↓) / 주 포지션 |
| 주 포지션 | 포지션 이모지 + 한글명 (미확인 시 "애매") |
| 플레이스타일 태그 | 높은 KDA / 팀 기여형 / 파밍 지향 / 딜 지향 / 탱커형 / CC 활용 / 시야 관리 / 힐·보호 / 정글 경험 / 밸런스형 |
| AI 팩폭 코멘트 | 빨간 헤더 "⚡ AI 팩폭 분석" + 어두운 빨간 배경 박스 |

### 6-4. 결과 화면 — 추천 포지션 카드

| 요소 | 내용 |
|------|------|
| 1순위 뱃지 | 금색 "1순위 추천" 뱃지 (1순위 카드만) |
| 포지션명 | 이모지 + 한글명 |
| 적합도 점수 | 우측 상단 금색 숫자 |
| 점수 바 | 금색 그라데이션 progress bar |
| 추천 이유 | 분석 기반 bullet point 목록 |
| 추천 챔피언 | 챔피언 이미지(Data Dragon) + 이름 + 추천 이유 |

## 7. 포지션 점수 계산 기준

| 포지션 | 주요 지표 |
|--------|----------|
| 탑 | 피해 흡수량, CC 시간, CS/분, KDA |
| 정글 | 킬 관여율, 몬스터 처치, 드래곤/바론 테이크다운, 오브젝트 스틸 |
| 미드 | 딜량, CS/분, KDA, 킬 관여율, 골드 |
| 원딜 | CS/분, 딜량, KDA, 골드, 피해 흡수량(낮을수록 유리) |
| 서포터 | 시야 점수, 킬 관여율, CC 시간, 힐량, 피해 흡수량 |

---

## 8. 플레이스타일 태그 판단 기준

| 태그 | 조건 |
|------|------|
| 높은 KDA | 평균 KDA ≥ 3 |
| 팀 기여형 | 평균 킬 관여율 ≥ 60% |
| 파밍 지향 | CS/분 ≥ 7 |
| 딜 지향 | 평균 딜량 ≥ 20,000 |
| 탱커형 | 평균 피해 흡수량 ≥ 25,000 |
| CC 활용 | 평균 CC 시간 ≥ 20초 |
| 시야 관리 | 평균 시야 점수 ≥ 25 |
| 힐/보호 | 평균 아군 힐량 ≥ 2,000 |
| 정글 경험 | 평균 몬스터 처치 ≥ 50 |
| 밸런스형 | 위 조건 해당 없음 |

## 9. 예외 처리

| 상황 | 처리 방식 |
|------|----------|
| 소환사 없음 | "소환사를 찾을 수 없습니다" 에러 메시지 |
| 게임 기록 없음 | "최근 게임 기록이 없습니다" 에러 메시지 |
| Riot API Rate Limit | 5개씩 배치 처리 + 1.2초 딜레이 |
| Claude API 과부하 | 1.5초 간격 최대 3회 재시도, 실패 시 코멘트 없이 결과 표시 |
| 포지션 판단 불가 (INVALID 등) | "애매"로 표기, 멀티 포지션으로 분류 |
| Riot API 키 만료 | 401 에러 메시지 표시 |
